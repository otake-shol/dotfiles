# /test - テストコード生成・レビュー

## テスト生成モード

指定されたコードまたは仕様に対して、包括的なテストケースを生成します。

### カバーすべき観点
- 正常系（Happy path）
- 異常系（Error cases）
- 境界値（Boundary conditions）
- エッジケース

### 遵守事項
- `expect(true).toBe(true)` のような意味のないアサーションは禁止
- 各テストケースは具体的な入力と期待出力を検証
- モックは必要最小限に
- テストケース名は何をテストしているか明確に記述

## テストレビューモード

既存のテストコードをレビューし、以下を確認：
- カバレッジの漏れ
- アサーションの妥当性
- モックの適切性
- テスト名の明確さ
