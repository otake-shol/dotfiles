# /spec - 仕様レビュー・作成

指定された仕様ファイルをレビューするか、新しい仕様を作成します。

## レビューモード（ファイル指定時）

以下の観点でレビューしてください：
- 曖昧な点・複数解釈可能な箇所
- 不足している情報（入力/出力/エラー）
- エッジケースの漏れ
- セキュリティ上の懸念
- パフォーマンス要件の有無

## 作成モード（要件指定時）

以下の形式で仕様を作成してください：

1. **概要** - 機能の目的
2. **ユーザーストーリー** - ユーザー視点での要求
3. **受け入れ基準** - チェックリスト形式
4. **入力/出力** - データ形式、バリデーション
5. **エラーケース** - 想定されるエラーと対処
6. **エッジケース** - 境界条件
7. **セキュリティ考慮事項** - 認証、認可、データ保護
