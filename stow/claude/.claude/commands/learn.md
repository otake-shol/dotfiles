# /learn - 学びの記録

PRレビューやデバッグから得た学びを「間違いログ」に記録します。

## 使い方

1. 問題を簡潔に説明してください
2. どのような対策を取るべきか教えてください

## 記録形式

CLAUDE.mdの「間違いログ」テーブルに追記:

```markdown
| 日付 | 問題 | 対策 |
|------|------|------|
| YYYY-MM-DD | 問題の概要 | 対策内容 |
```

## 例

- 「非同期処理でawaitを忘れてバグになった」→ 非同期関数呼び出しは必ずawaitを確認
- 「型定義が緩くて実行時エラー」→ strict: trueを徹底
